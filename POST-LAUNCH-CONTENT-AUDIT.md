# Post-Launch Content Audit

This is the working content audit for the current `wuwabanners.net` version after the latest release.

## Audit Goal

The goal is not to score every page. The goal is to decide:

- what is already structurally strong
- what is still too template-like
- what should be strengthened before adding lots of new URLs

## What Is Working Well

### 1. Site structure is now coherent

The site no longer feels like a flat pile of SEO pages. The strongest improvement is structural:

- top nav is clear
- homepage acts like a front door
- hubs work as list pages
- detail pages exist under the main branches
- support pages sit below the main details

This is the correct long-term shape for the project.

### 2. Homepage now carries the right priorities

The homepage already does the right jobs first:

- current timeline snapshot
- current banner end timing
- next banner start timing
- pull path
- section entry points
- branch explanation

That means new users can understand the site without guessing where to click first.

### 3. Banner history is much stronger than before

`banner history` is now one of the best-built parts of the site because it has:

- list page + detail page structure
- visual lineup cards
- phase-to-phase navigation
- stronger internal links back into current / next / rerun pages

This branch is already close to the correct long-term pattern for the rest of the site.

### 4. Reference branches are usable

`characters`, `weapons`, and `items` now have:

- list pages
- image-driven browsing
- detail pages
- next-step links

That is enough to support browsing intent, even if not all detail pages are deep yet.

## Main Weaknesses To Fix Next

### 1. Character support pages are still framework-heavy

The `materials / build / team comps` pages for focus characters are structurally sound, but many of them still read like strong templates rather than fully mature guides.

The biggest missing layer is concrete decision value:

- real materials lists
- real pre-farm split
- real build priority logic
- real team shell examples

These pages should become more specific before the site expands much wider.

### 2. Pull advice hub still needs stronger comparison blocks

`pull-advice` works as a hub, but it should eventually do more direct comparison work:

- current phase vs next phase
- urgent pull now vs safe save later
- role overlap
- pity and weapon pressure

At the moment it routes well, but still leaves too much of the actual decision to the child pages.

### 3. Some source chains are still mixed

The site source policy is official-first, but some history rows still use secondary source references.

This is not a site-breaking issue, but it should be cleaned up over time, especially for:

- old phase timing
- old banner lineup confirmation

### 4. Reference detail pages are structurally better than they are informationally deep

The `characters / weapons / items` detail pages now browse well, but many still need a richer reason to exist beyond:

- visual confirmation
- next-step links

That is acceptable for now, but later they should gain:

- cleaner classification
- more useful role / use-case framing
- stronger context from the planning side of the site

## Highest-Priority Page Group To Strengthen

Do not try to deepen the whole site at once.

The best next content pass is this focused group:

### Tier 1

- `/wuthering-waves-next-banner/`
- `/wuthering-waves-current-banner/`
- `/pull-advice/`
- `/wuthering-waves-banner-history/`

### Tier 2

- `/wuthering-waves-hiyuki-materials/`
- `/wuthering-waves-hiyuki-build/`
- `/wuthering-waves-hiyuki-team-comps/`
- `/wuthering-waves-denia-materials/`
- `/wuthering-waves-denia-build/`
- `/wuthering-waves-denia-team-comps/`

These are the pages most likely to improve both usefulness and internal linking strength.

## Best Next Content Improvements

### Improvement 1: make support pages more concrete

For focus characters, the next pass should add:

- exact material buckets
- what is safe to pre-farm vs what must wait
- fallback weapon route vs premium route
- one quick team shell for low-commitment planning

### Improvement 2: make pull-advice more comparative

Add clearer modules for:

- current phase best for X accounts
- next phase best for Y accounts
- when saving is stronger than spending now

### Improvement 3: replace weak secondary source residue

Where official archive or official event references can be found, replace weaker history source rows over time.

### Improvement 4: keep reference branches supporting, not distracting

`characters / weapons / items` should remain useful support branches, but they should not pull attention away from the higher-value banner and planning branches.

## Suggested Work Order

1. Keep using the current structure.
2. Do not mass-produce new long-tail URLs for now.
3. Strengthen the Tier 1 and Tier 2 pages above.
4. Only then expand further history or reference coverage.

## Release Readiness Verdict

Current verdict:

- structure: strong enough
- update workflow: nearly strong enough, with automation now added
- content depth: mixed, but good enough to keep iterating in public

The site should now shift from `structure-building` into `core-page deepening`.
